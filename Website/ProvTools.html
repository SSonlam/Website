Data Provenance Tools

Data Provenance is supported in many ways. Especially in today's job industry there is a demand for tools to be able to handle data. Some of the most popular ones are as follows:

Git

Git is specifically designed for code and software engineering, the technology promotes software team work and software reuse. It has features such as creating a space for a team to work on the same files, “pushing,” which is when team members make edits to code, and “pulling,” when team members receive the code from their project.
  • Designed for code and software engineering
  • Promotes software reuse
  • Allows teams to work on the same code file, saving any changes made to the code

PRiME

PRiME is a software engineering technique for adapting application designs to enable them to interact with a provenance middleware layer [1].
  • Software engineering technique for adapting application designs to enable them to interact with a provenance middleware layer
D-PROV
D-PROV is a model in which provenance traces of scientific workflow runs are captured and stored alongside the data products [2].
  • A model in which provenance traces scientific workflow are captured and stored along with the data products
