Importance of Data Provenance in the Software Field

Who needs to use provenance?

  • People related to the project or the product. For example, they could be developers, testing teams, managers, software design architecture, and even stakeholders. [1]

Why is provenance needed?

  • To understand the project thoroughly, developers and stakeholders would want to know about its features and the change history for comparison.
  • In order to build a quality product, developers need to know where their data came from to determine its reliability and quality. For example, when developers maintain or evolve an existing code, they would want to know when the code was built, any existing bugs, what generation the code is, which features and requirements are needed, and so on. [1] Developers might also want to know the extent to which the code was tested. Was the code tested every time it was modified? The number of unit tests give a hint on where the quality of the code is. If the code were to suddenly fail, current developers might want to contact the responsible developer because they would have the most knowledge to fix the problem (They might have fixed the problem already in a previous test).

When/where is provenance needed?

  • Provenance is used to determine whether code/data is trustworthy by looking at the believability of the source, reasonableness of data, and temporality of data.

    • Believability of source: the extent to which a data value from a trustworthy source
    • Reasonableness of data: The extent to which a data value is likely
    • Temporality of data: the extent to which data value is credible based on transaction and valid times
Some cases where provenance is needed to search and store data.
  • In the writing requirements process, besides the defined requirements from stakeholders, developers can search for similar previous projects (bugs and risks) to improve the requirement sets.
  • While developing the project, developers usually search for available code instead of writing the whole project from scratch. The code could be in the library, open sources, or product from third parties. The provenance of that code is important to determine how reliable and well tested that code is.
  • When the project is passed to the testing teams, they need to know where the data, resources, and code came from to build the appropriate test cases.
  • When the project is complete and sent to stakeholders, they might want to search for previous related projects to compare and analyse its quality.
  • All data about the project should be stored in the system in case developers want to maintain or evolve the project. Sometimes the maintaining developer team is not the developing team, so this data is very important for new team developers to understand the project.
  • Data provenance is also needed when input data is being used by a software. Data provenance is used to determine the origin and transmission history of the data. Organizations may use blockchain technology for authenticating data. A distributed ledger provides a verified chain of transaction records which makes data within the chain easy to trace. Data within the chain can not be modified without approval.
