Importance of Data Provenance in the Software Field

Why is it Needed? (more detail is shown in part 3, Question “Why is provenance needed?”)>

  • Important for developers and stakeholders who want to know more about features and their change history
  • Determines the reliability and quality of the data being used
  • When developers are using existing code for their projects they would want to know the following

When is it used?

  • Data provenance is used during the development of a project
  • Testing process
  • After the project is complete stakeholders might want to search for previous projects, similar to the new one, to compare and analyze its quality
  • All data should be stored for future team developers to understand the project
